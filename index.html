HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Headers are used 1-6, the first should be used only once.
  • Ul command is used for bulletted lists while ol is used for numbered lists. Li is used within these bodies.

DAY 2 HTML NOTES

  • HREF is used for URLS typically.
  • Src is used primarily for images (can be a link to an image) and javascripts.
  • Id is used for a single element
  • Class is used to classify and can be used for multiple elements.
  • When bug fixing and troubleshooting, its best to start from the top and work down. For example, when troubleshooting why CSS styles are not appearing on the HTML file, it would be best to start by checking if the CSS file is actually linked to the HTML file correctly.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The "." is a designator for class selector.
  • Color can be changed in multiple way, the most specific by use of a hexadecimal value for the desired color.

DAY 2 CSS NOTES

  • . is used to signify class attributes.
  • # is used to signify id attributes
  • To link the CSS document to the HTML document, need to use the LINK tag in the head tag and be sure to use the "rel" attribute and specify "stylesheet" for the relationship and then use the href attribute for the actual file location.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b "branch-name": creates a new branch and switches to it.

GIT COMMANDS FOR PUSHING AND PULLING

    GIT PULL

  • git status [check the status of the branch you are in]
  • git pull [branch name here] (pulls the called out branch (usually the main branch) from the server to the designated area called out on the machine)

    • GIT PUSH

  • git add [name files here] (can use "add ." to select all files)
  • git commit -m "message of the commit located here"
  • git push [branch name here] (pushes the origin main from the machine to the server)
  • git status to check that everything is up to date and pushed successfully

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Arrays are useful tools for storing multiple items with different data types in a single variable, but not all arrays must have multiple data types.
  • Variables using the "const" keyword, which "locks down" the value of the variable so that it can't be reassigned.
  • Variables usiung the "let" keyword allow for the value to be reassigned (useful for for loops and iterations).
